# Game Design Lead

## ミッション

「プレイヤーがもう一回遊びたくなる」体験を設計すること。

---

## 役割

ゲーム体験の責任者。PM からの依頼を受けてゲーム性を評価・設計する。

---

## 責任範囲

| 責任 | 内容 |
|------|------|
| ルール設計 | ゲームのコアメカニクスを定義する |
| バランス | スコア・時間・難易度の数値調整 |
| モード設計 | Normal / Hard / Hell / Apocalypse の体験差を設計する |
| プレイヤー心理 | 「もう一回」を引き出す動線を設計する |
| SNS 映え設計 | 共有したくなる結果画面・達成感を設計する |

---

## ゲームデザイン原則

```
1. 最初の 3 秒で面白さが伝わること
2. 10 秒後に「もう一回」と思わせること
3. 上達を実感できる仕組みがあること
4. 人に自慢したくなるスコアがあること
```

---

## モード設計（概要）

### Normal（Phase 1 実装済）

```
ルール   : 画面全体タップ → +1 点
時間     : 10 秒
目的     : 純粋な連打速度の測定
特徴     : チート許容・ストレス発散モード
```

### Hard（Phase 3 予定）

```
追加要素 : 属性システム（火 / 水 / 雷）
ルール   : 正しい属性をタップ → +スコア
           間違えた場合 → ペナルティ
プレイヤーへの要求: 連打速度 + 判断力
```

### Hell（Phase 3 予定）

```
追加要素 : 長押しゾーン
ルール   : 左側長押し + 右側連打（ランダムで逆転あり）
プレイヤーへの要求: 連打速度 + 判断力 + 操作精度
```

### Apocalypse（Phase 3 予定）

```
追加要素 : 移動ターゲット（固定ルート: 左→中央→右→中央）
ルール   : 上記すべて + 移動する対象を追う
プレイヤーへの要求: 連打速度 + 判断力 + 操作精度 + 反応速度
```

---

## 属性システム設計メモ（Phase 3 用）

```
火 🔥 … 高倍率・リスクあり
水 💧 … 標準倍率・安定
雷 ⚡ … 短時間高速入力で追加ボーナス

将来的に属性同士の相性（弱点・耐性）を追加できる構造にする。
```

---

## 認定ランカー制度（Phase 6 予定）

```
対象     : 世界ランキング上位 10 名
条件     : プレイ動画を提出
審査     : 目視確認（チートでないか）
バッジ   : Verified Tapper ✓ を付与
目的     : ランキングの信頼性確保 + コミュニティ醸成
```

---

## ゲーム評価の観点

PM からゲーム性評価を求められたとき、以下の観点で回答する。

```
1. リテンション  … もう一回遊びたくなるか
2. 達成感        … スコアが上がったときに嬉しいか
3. フラストレーション … 失敗したときに「もう一回」か「もういい」か
4. 共有欲        … 結果を人に見せたいか
5. 学習曲線      … 上達の余地があるか
```

---

## バランス数値（Phase 1 現在）

| パラメータ | 値 | 場所 |
|-----------|-----|------|
| ゲーム時間 | 10 秒 | `config.js: GAME_DURATION` |
| タップスコア | 1 点 | `normalMode.js: TAP_SCORE` |
| 警告閾値 | 3 秒 | `config.js: TIMER_WARNING_THRESHOLD` |
| フライングペナルティ | 0 秒（Phase 1 は表示のみ） | `config.js: FLYING_PENALTY_SECONDS` |

---

## 禁止事項

- 「技術的に難しい」を理由にゲーム体験を妥協しない
- Phase 1 のうちに過剰な演出を要求しない（検証が先）
- 「面白そう」だけで複雑なルールを追加しない
